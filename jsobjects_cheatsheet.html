Basic Objects

There are TWO kinds of Objects in JS. One refers to Javascript’s built-in list of objects and any instances of them.

Some examples are:

  • Math
  • Dates
  • Regular Expressions
  • Functions
  • Strings
  • Numbers
  • Boolean

The last three are actually primitives – you don’t create any instances explicitly in JS. You set variable_name = “” instead of explicitly creating a string instance. The interpreter then converts the primitive to a method, calls its method as an object and then revers the value back to a primitive.

Object Literals

Object Literals are the second type of objects and are denoted by curly braces.

var object = {};
TIP FOR RUBYISTS: This is your HASH equivalent! The only difference is that instead of key => value, in JS it is written as property_name : value.

Key-Value pairs are known as Property names & Values. Property names can be strings or identifiers, while values can be any valid expression. Each pair is comma delimited.

var object = { foo: 1, "bar": "some string", baz: { } };

Accessing Properties

dot notation - a property is accessed by giving the host object’s name, followed by a period (or dot), followed by the property name.

object.foo = object.foo + 1;

Dot notation has to be followed by the property name, with no quotation marks, even if it is written as a string in the object literal. You can’t use the dot notation with variables.

In bracket notation, the object name is followed by a set of square brackets. You can put either code, variables or a property name. Code is evaluated and turned into a string. Explicit property names must be given as strings

object["foo"] = object["foo"] + 1;

Properties of nested objects can be accessed by chaining dot and/or bracket references together. (you can use the two different types together too)

object["baz"]["foo"]["bar"];
TIP FOR RUBYISTS: functions can be values in a property name – value pair in an object literal. They can be called using their property name, followed by any parameters that their method may have:
var object = { sum: function(foo, bar) { return foo + bar; } }; object.sum(1, 2); object["sum"](1, 2);

Manipulating Data in Object Literals

Adding new data to an object works the same way as creating an assignment statement:

object.foo = 1;

Get the value using the keyword: alert

alert(object.name) —> “value";

Delete the pair using the keyword delete

delete (object.name);

Arrays

Arrays are a type of Object literal specialized for storing sequences of things. Arrays are written as a list of values between square brackets, with each element separated by commas.

var listOfNumbers = [2, 3, 5, 7, 11]; console.log(listOfNumbers[1]); // → 3

Array Methods

.push adds to end
.join flattens out into a string
.popremoves from end
unshiftadd to the start
shift remove from the start
indexofsearches for a given element, starting from the beginning and going to the end. returns the first instance*
lastindexof searches for a given element but starts from the back.*
slicetakes a start and an end index and returns an array that has only the elements between those indices. The start index is inclusive, the end index exclusive
concat can be used to glue arrays together, similar to what + does for strings

*this method can take an optional parameter to tell it at what index to start looking

TIP FOR RUBYISTS: note that console.log writes the array out as an array, more like ‘p’ would, not like ‘puts'

LOOPING THROUGH OBJECT LITERALS

Javascript’s main implementations of looping are FOR and WHILE loops

For loops use the following format:

The first section refers to the counter’s starting value. The second refers to the condition under which the code will continue to loop, and the third is how the counter will increment after each loop through the code that follows inside curly braces.

for (i = 0; i < object.length; i++) { some code };

Javascript has a special syntax for iterating through object literals using the keyword in.

It uses the following syntax:

for (property in object literal) { do something to each object };
