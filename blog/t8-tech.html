Regular expressions are special string based codes used to describe searching patterns within a file, database or other document.A regular expression, also known as a 'regex' or 'regexp', is a PATTERN that describes a certain amount of text It transcends coding languages, and can be used in Ruby, Javascript, Python and more.

A regular expression can be as simple as literal text, (ie: 'blue'). When you search these patterns you get MATCHES. A match is a piece of text, sequence of bytes or group of characters that the regex processing software found to match the regular expression's pattern. The regex processing software refers to the 'engine' that can PROCESSS the regexes, such as Ruby or Python. Although the exact way of callling regexes might vary from program/language to language, the regular expressions themselves stay the same.

The reg ex patterns we are searching are not always as simple as trying to find 'blue'. Most of the time, our searches are more complex and the regex ends up looking more like this: \b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b Believe it or not, this complicated string is to search for email addresses.

I have some basic facts to start searching below.

  1. Regular expressions are started and ended wtih forward slashes: /. The 'search codes' go in between.
  2. Single Character: The most basic type of regex is a single letter, like 'a'. If you search for 'a' in 'Jack and Jill', it will return the first occcurrence in the string: the 'a' in Jack.
  3. Character Pairs: Let's say you want to see an instance of TWO(or more) possible letters. [ab] would return the first instance of EITHER a OR b, whichever appeared first
  4. Character Ranges: You can specify ranges or even sets of ranges with a hyphen. [0-9] gives you the first digit, [0-9a-z] gives you the first lowercase letter or number. Uppercase and lowercase letters are considered separate.
  5. A single character followed by [^x] will give you any set of two characters that start with the given single character
  6. [^a-z] gives you a range of characters to IGNORE. The above would give you uppercase letters, numbers, spaces and symbols.
  7. Use parentheses to get GROUPS of characters (as opposed to square brackets which has them as options). (abc) will only return instances that
  8. \d matches any single digit
  9. \w matches any single letter
  10. \s matches any single whitespace
  11. \b refers to word boundaries
  12. . matches any single character except a line break, but is used less on its own and more in the middle of a word with a variable letter: like gr.y which could return gray, grey, gr!y, etc.
  13. The caret or anchor (^) stands for the start of the line. It indicates that we are interested in a pattern from the very beginning of a given line.
  14. Based on the regex engine you are using, there are different ways to look up multiple instances. In text editor,for instance, you have 'next' and 'replace'. But what if you want to look up multiple instances without the 'next' button or its equivalent?
    • a* gives you 0 or more instances
    • a+ gives you 1 or more instances
    • a? gives you 0 or 1 instances
    • a{#}, with # replaced by any number, gives you exactly that number of instances
    • a{1,3} gives you between 1 and 3 instances
    • a+? will match as few as possible
  15. You can also use | as 'OR', to match one combo OR another.

    16. Let's go back to that scary looking regex from the beginning of this post. \b[A-Z0-9._%+-]+@[A-Z0-9.-]+\.[A-Z]{2,4}\b. We can now break this down and decipher it.

    • \b refers to word boundaries
    • the first set between brackets uses RANGES and SETS. It says look for A-Z, digits 0-9, and the symbols (. % + -). The second set follows the same pattern.
    • the + spreads the search so it will return as many consecutive characters that meet the search as possible until it encounters something that doesn't.
    • The @ just refers to the @ symbol from an email.
    • [A-Z]{2,4} looks for groups of capital letters that are between 2 and 4 characters long
